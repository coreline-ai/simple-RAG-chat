## 1. 제품 개요

### 1.1 제품명
**RAG-Text-LLM** — 정형 데이터 기반 지식 검색 및 질의응답 시스템

### 1.2 제품 목적
조직 내 채팅 로그(정형 데이터)를 벡터화하여 저장하고, 자연어 질의를 통해
관련 대화를 검색한 후 로컬 LLM으로 답변을 생성하는 RAG 시스템.

### 1.3 핵심 가치
| 가치 | 설명 |
|------|------|
| **완전 로컬 실행** | 외부 API 키 불필요, 데이터가 서버 밖으로 나가지 않음 |
| **정형 데이터 최적화** | 비정형 텍스트 RAG와 차별화된 메타데이터 기반 검색 |
| **스마트 쿼리 라우팅** | 질의 의도를 자동 분류하여 최적 검색 전략 선택 |
| **제로 설정** | Docker/PostgreSQL 없이 Python + Ollama만으로 실행 |

### 1.4 대상 사용자
- 사내 채팅 로그를 검색/분석해야 하는 팀 리더
- 회의록/대화 기록에서 특정 정보를 찾아야 하는 직원
- 로컬 환경에서 RAG 파이프라인을 학습하려는 개발자

## 3. 비기능 요구사항

### 3.1 성능

| 항목 | 목표 | 현재 |
|------|------|------|
| 쿼리 분석 속도 | < 10ms | ✅ < 1ms (규칙 기반) |
| 벡터 검색 속도 | < 500ms | ✅ ~200ms |
| LLM 응답 시간 | < 30초 | ⚠️ 15~210초 (로컬 7B 한계) |
| 임베딩 처리량 | > 100건/분 | ✅ ~550건/분 |
| 동시 사용자 | > 5명 | ⚠️ 1~2명 (Ollama 병목) |

### 3.2 데이터

| 항목 | 사양 |
|------|------|
| 지원 데이터 형식 | `[날짜, 시간, 채팅방이름, 입력내용, 사용자]` |
| 최대 데이터 건수 | 10,000건 (테스트 완료), 100,000건 (이론상 가능) |
| 임베딩 차원 | 1024 (bge-m3) |
| 메타데이터 필드 | room, user, date, date_int, time, original |

### 3.3 보안/프라이버시

| 항목 | 사양 |
|------|------|
| 데이터 저장 | 로컬 파일 시스템만 (외부 전송 없음) |
| 인증 | 없음 (로컬 전용 설계) |
| API 키 | 불필요 (모든 모델 로컬 실행) |

### 3.4 호환성

| 항목 | 사양 |
|------|------|
| Python | 3.11+ |
| OS | Windows, macOS, Linux |
| 필수 외부 서비스 | Ollama (로컬) |

## 4. 데이터 형식 명세

### 4.1 입력 데이터
```
[2024-02-21, 14:30:00, 백엔드개발, 서버 배포 3차 완료했습니다, 김민수]
```

| 필드 | 타입 | 형식 | 예시 |
|------|------|------|------|
| 날짜 | 문자열 | YYYY-MM-DD | 2024-02-21 |
| 시간 | 문자열 | HH:MM:SS | 14:30:00 |
| 채팅방이름 | 문자열 | 자유 | 백엔드개발 |
| 입력내용 | 문자열 | 자유 | 서버 배포 3차 완료 |
| 사용자 | 문자열 | 한글 이름 | 김민수 |

### 4.2 임베딩 데이터
```python
{
    "embedding_text": "백엔드개발 김민수: 서버 배포 3차 완료했습니다",
    "original": "[2024-02-21, 14:30:00, 백엔드개발, 서버 배포 3차 완료했습니다, 김민수]",
    "metadata": {
        "room": "백엔드개발",
        "user": "김민수",
        "date": "2024-02-21",
        "date_int": 20240221,
        "time": "14:30:00"
    }
}
```

## 6. 질의 유형 지원 매트릭스

| 질의 유형 | 예시 | 분석 결과 | 검색 전략 |
|----------|------|----------|----------|
| 내용 검색 | "서버 배포 관련 대화" | intent=search, 필터 없음 | vector |
| 날짜 검색 | "2024-02-21 일어난 일" | date=2024-02-21 | hybrid |
| 날짜 범위 | "2024년 3월 대화" | date_from~to | hybrid |
| 채팅방 검색 | "AI연구팀 대화" | room=AI연구팀 | metadata |
| 사용자 검색 | "김민수가 뭐 했어?" | user=김민수 | hybrid |
| 목록/나열 | "AI연구팀 팀원 이름" | intent=list, room=AI연구팀 | aggregate |
| 통계/집계 | "가장 활발한 채팅방" | intent=aggregate | aggregate |
| 복합 필터 | "3월 개발팀 배포 현황" | room+date_range+keyword | hybrid |
| 상대 날짜 | "최근 대화", "이번 주" | date_from~to (7일) | hybrid |
| 요약 | "어제 무슨 일이 있었나" | intent=summary | hybrid |

## 7. 제약사항 및 알려진 한계

| ID | 제약사항 | 영향 | 우회 방법 |
|----|---------|------|----------|
| C-01 | LLM 응답 시간 15~210초 | 사용자 경험 저하 | 더 작은 모델 또는 클라우드 API |
| C-02 | ChromaDB $gte/$lte 숫자만 지원 | 날짜 범위 비교 | date_int 정수 필드로 우회 |
| C-03 | Ollama 단일 요청 처리 | 동시 사용자 제한 | 큐잉 또는 복수 인스턴스 |
| C-04 | 인증/인가 없음 | 로컬 전용 | 프로덕션 시 미들웨어 추가 |
| C-05 | aggregate 전략 메모리 전체 로드 | 대규모 데이터 OOM | 페이징 처리 |
| C-06 | 한국어 이름 감지 오탐 가능 | "이번에" 등 일반 단어 | 블랙리스트 확장 |
