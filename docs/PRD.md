**문서 버전**: 1.1
**작성일**: 2026-03-27
**상태**: 현재 구현 반영

## 1. 제품 개요

### 1.1 제품명
**Simple RAG Chat** — 정형 채팅 로그와 이슈 데이터를 위한 지식 검색 및 질의응답 시스템

### 1.2 제품 목적

조직 내 채팅 로그와 이슈 관리 데이터를 구조화된 메타데이터와 함께 저장하고, 자연어 질의로 관련 기록을 검색한 뒤 답변을 생성하는 로컬 우선 RAG 시스템이다.

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| 로컬 우선 실행 | Ollama-only 모드는 API 키 없이 동작하고, 필요 시 프록시/gateway LLM으로 확장 가능 |
| 정형 데이터 최적화 | 날짜, 채팅방, 사용자, 담당자, 상태 같은 필터를 검색에 직접 활용 |
| 스마트 쿼리 라우팅 | 질의 의도를 자동 분류해 `vector`, `metadata`, `hybrid`, `aggregate` 전략을 선택 |
| 운영 단순성 | FastAPI + ChromaDB + 단일 페이지 UI로 빠르게 실행 가능 |

### 1.4 대상 사용자

- 사내 채팅 기록을 검색하고 요약해야 하는 팀 리더
- 이슈 엑셀 데이터를 분석해야 하는 운영/개발 담당자
- 로컬 환경에서 RAG 파이프라인을 학습하거나 실험하려는 개발자

## 3. 비기능 요구사항

### 3.1 성능

| 항목 | 목표 |
|------|------|
| 쿼리 분석 속도 | < 10ms |
| 벡터 검색 속도 | < 500ms |
| 첫 응답 체감 시간 | SSE 사용 시 가능한 한 빠른 첫 토큰 전송 |
| 임베딩 안정성 | timeout/5xx/429에서 bounded retry |
| 동시 임베딩 요청 | `EMBEDDING_MAX_CONCURRENCY`로 제한 가능 |

### 3.2 데이터

| 항목 | 사양 |
|------|------|
| 지원 데이터 형식 | `.txt`, `.xlsx`, `.xls` |
| 저장소 | ChromaDB + `documents.json` + `embedding_cache.db` |
| 메타데이터 필드 | room, user, assignee, status, date, date_int 등 |
| 캐시 | SQLite 임베딩 캐시 + 인메모리 질의 캐시 |

### 3.3 보안/프라이버시

| 항목 | 사양 |
|------|------|
| 데이터 저장 | 로컬 파일 시스템 중심 |
| 인증 | 기본 제공 없음 |
| 외부 연결 | Ollama는 필수, 프록시/direct는 선택 |

### 3.4 호환성

| 항목 | 사양 |
|------|------|
| Python | 3.11+ |
| OS | Windows, macOS, Linux |
| 필수 외부 서비스 | Ollama |
| 선택 외부 서비스 | OpenAI 호환 프록시, Codex direct backend |

## 4. API 명세

### 4.1 질의 요청

```json
POST /query
{
  "question": "2025년 3월에 Sujin이 처리한 이슈는?",
  "top_k": 5
}
```

### 4.2 질의 응답

```json
{
  "question": "2025년 3월에 Sujin이 처리한 이슈는?",
  "answer": "2025년 3월 Sujin 담당 이슈는 ...",
  "sources": [
    {
      "id": "doc_line_42",
      "chunk_index": 42,
      "content": "[이슈] GPU 메모리 부족 발생 ...",
      "score": 0.8542
    }
  ]
}
```

### 4.3 스트리밍 응답

- 엔드포인트: `POST /query/stream`
- 형식: `text/event-stream`
- 이벤트 payload:
  - 토큰 이벤트: `{ "token": "...", "done": false }`
  - 완료 이벤트: `{ "done": true, "sources": [...] }`
  - 오류 이벤트: `{ "error": "...", "done": true }`

## 5. 질의 유형 지원 매트릭스

| 질의 유형 | 예시 | 분석 결과 | 검색 전략 |
|----------|------|----------|----------|
| 내용 검색 | `서버 배포 관련 대화` | keyword 중심 | `vector` |
| 날짜 검색 | `2025-03-01 일어난 일` | `date=2025-03-01` | `hybrid` |
| 날짜 범위 | `2025년 3월 이슈` | `date_from~to` | `hybrid` |
| 채팅방 검색 | `AI연구팀 대화` | `room=AI연구팀` | `metadata` |
| 담당자 검색 | `Sujin 담당 이슈` | `assignee=Sujin` | `metadata` |
| 목록/나열 | `완료된 이슈 목록` | `intent=list` | `aggregate` |
| 통계/집계 | `가장 많은 이슈를 처리한 담당자` | `intent=aggregate` | `aggregate` |
| 복합 필터 | `3월에 Sujin이 완료한 이슈` | 날짜+담당자+상태 | `hybrid` |
| 상대 날짜 | `최근 이슈`, `이번 주` | 최신 데이터 기준 날짜 범위 | `hybrid` |

## 6. 제약사항 및 알려진 한계

| ID | 제약사항 | 영향 | 우회 방법 |
|----|---------|------|----------|
| C-01 | 임베딩은 항상 Ollama 의존 | 프록시 모드만으로는 전체 파이프라인 실행 불가 | `bge-m3` 설치 및 Ollama 실행 |
| C-02 | aggregate 전략은 결과를 메모리로 읽는다 | 대규모 데이터셋에서 메모리 사용량 증가 가능 | 페이징/사전 집계 추가 |
| C-03 | 인증/인가 없음 | 로컬/내부망 용도에 적합 | 운영 배포 시 별도 미들웨어 추가 |
| C-04 | Chroma 및 documents 저장소는 파일 쓰기 권한이 필요 | 읽기 전용 경로에서는 업로드 실패 | `CHROMA_PERSIST_DIR` 권한 확인 |
| C-05 | Ollama 응답 시간은 모델 크기에 영향받음 | 사용자 체감 지연 가능 | 더 작은 모델 또는 프록시 LLM 사용 |
